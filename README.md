## 一、项目概述

### 1.1 项目定位

《盗梦空间》是一款 **2D 90° 顶视** 像素风格的不对称竞技塔防游戏，灵感来源于《盗梦空间》的多层梦境设定。玩家可选"人类"或"猎梦者"两个阵营，体验完全不同的目标、视角和操作。

### 1.2 阵营与玩法

| 阵营 | 目标 | 操作 | 胜利条件 |
|------|------|------|----------|
| 人类模式 (`MODE_HUMAN`) | 建造防御、击退猎梦者 | WASD 控制玩家人类，其余 5 人为 AI 盟友 | 击杀猎梦者 |
| 猎梦者模式 (`MODE_HUNTER`) | 攻破防御、消灭所有人类 | WASD 控制猎梦者，6 个人类全部为 AI | 击杀所有人类 |

### 1.3 项目特点

- **双阵营对抗**：视角、策略体系与胜利条件截然不同
- **4 种地图**：简单 / 苔藓 / 雪地 / 炼狱，**全部统一 33×39 矩阵描述**，候选池 = `[easy, normal, snow, hell]`
- **零图片资源**：所有纹理通过 Pygame 绘制 API 程序化生成
- **完整 AI 对战**：人类和猎梦者均支持 AI 单机对战
- **5 种猎梦者**：晴天娃娃 / 大头 / 孙婆婆 / 绷带 / 红裙小女孩，各有独特技能
- **6 + 1 难度等级**：简单 → 普通 → 困难 → 噩梦 → 地狱 → 炼狱 + 无尽模式（通关炼狱后解锁）
- **20 种建筑 × 多级升级** + 电力系统 + 经济管理 + 路径选择
- **门滑入动画通用化**：所有地图都支持"人类踏入红格 → 蓝格门平滑滑入红格"的门关闭动画
- **代码规模**：约 **10,500+ 行 Python 代码**，13+ 个核心模块

## 二、技术栈与运行

### 2.1 技术栈

| 类别 | 选型 | 备注 |
|------|------|------|
| 编程语言 | Python 3.12 | 仅使用 `random` / `heapq` / `math` / `sys` 等标准库 |
| 游戏框架 | Pygame 2.6 | 唯一第三方依赖 |
| 中文字体 | `simhei`（系统黑体） | 缺失时自动回退到 `pygame.font.Font(None, …)` |
| 操作系统 | Windows | 已适配 `pygame.RESIZABLE` 窗口缩放 |

### 2.2 依赖安装与运行

```bash
# 安装唯一第三方依赖
pip install pygame>=2.6

# 启动游戏
python main.py
```

> 项目根目录未提供 `requirements.txt`，因为除 Pygame 外的所有 import 均为标准库。

### 2.3 操作键位

| 按键 | 功能 |
|------|------|
| `W` / `A` / `S` / `D` 或方向键 | 移动角色（4 格/秒） |
| 鼠标左键 | 建造 / 升级 / 切换视角（HUD 头像可切换跟随目标） |
| 鼠标拖拽 | 拖动摄像机视野（位移 > 5 px 时脱离自动跟随） |
| `F` | 玩家手动激活**屠龙刀**（无距离限制，优先砍正在攻击本房间门的猎梦者） |
| `P` | 暂停 / 继续 |
| `ESC` | 返回主菜单 |
| `SPACE` | 结算界面确认返回主菜单 |

## 四、核心架构

### 4.1 主循环

[main.py](main.py) 实现经典三段式循环（60 FPS，`dt` 钳制 0.1 秒上限）：

```python
while running:
    dt = clock.tick(60) / 1000.0
    # ① 事件处理 → InputRouter → 分发
    # ② 逻辑更新（按 phase 分支：SAFE / PLAYING）+ update_economy
    # ③ 渲染（按 phase 分支：MENU / 游戏中 / 结算）
    pygame.display.flip()
```

### 4.2 数据流

```
[玩家输入] → [InputRouter] → [main.py 分发]
                                ↓
                ┌───────────────┼───────────────┐
                ↓               ↓               ↓
          [GameState]      [Camera]      [UI/HUD]
                ↓               ↓
            [Entities]      [Renderer]
                ↓
            [Systems]
```

## 五、游戏状态机

```python
PHASE_MENU    = 0   # 主菜单
PHASE_INIT    = 1   # 地图/实体初始化（瞬时）
PHASE_SAFE    = 2   # 安全期（30 秒，玩家布局）
PHASE_PLAYING = 3   # 正式对抗
PHASE_VICTORY = 4   # 胜利结算
PHASE_DEFEAT  = 5   # 失败结算
```

**状态转换**：

```
PHASE_MENU → PHASE_SAFE → PHASE_PLAYING → VICTORY/DEFEAT
              ↑ 30秒倒计时
```

**胜负判定**（`check_win_lose()`）：

| 模式 | 胜利条件 | 失败条件 |
|------|----------|----------|
| `MODE_HUMAN` | 猎梦者 HP ≤ 0 | 玩家人类死亡 |
| `MODE_HUNTER` | 全部人类死亡 | 猎梦者 HP ≤ 0 |

### 5.1 难度等级与解锁

| 编号 | 常量 | 中文名 | 猎梦者 | 数量 | 解锁条件 |
|------|------|--------|--------|------|----------|
| 0 | `DIFF_EASY` | 简单 | 晴天娃娃 | 1 | 默认 |
| 1 | `DIFF_NORMAL` | 普通 | 大头 | 1 | 通关简单 |
| 2 | `DIFF_HARD` | 困难 | 大头/孙婆婆/绷带/红裙 | 1 | 通关普通 |
| 3 | `DIFF_NIGHTMARE` | 噩梦 | 同上 | 2 | 通关困难 |
| 4 | `DIFF_HELL` | 地狱 | 同上 | 3 | 通关噩梦 |
| 5 | `DIFF_PURGATORY` | 炼狱 | 同上 | 4 | 通关地狱 |
| 6 | `DIFF_ENDLESS` | 无尽 | 同上 | 1+ | 通关炼狱 |

> 存档保存在 [save.json](save.json)，由 [core/save_data.py](core/save_data.py) 读写。

### 5.2 无尽模式

- **初始猎梦者数量**：`ENDLESS_START_COUNT = 1`
- **每波新增猎梦者数**：`ENDLESS_COUNT_PER_WAVE = 1`
- **每波属性倍率递增**：`ENDLESS_STAT_MULT_PER_WAVE = 0.15`（+15%/波）
- 最高波数写入存档，作为玩家成就记录

## 六、地图系统

### 6.1 4 种地图规格（**全部统一 33×39 矩阵**）

| 地图 | `map_type` | 尺寸（列×行） | 走廊风格 | 房间数 | 颜色编码 |
|------|-----------|--------------|----------|--------|----------|
| 简单 (Easy) | `easy` | **33×39** | 木色 (`wood`) | 13 | 黄房/黑墙/白走廊/蓝门/红箭头/绿出生 |
| 苔藓 (Normal) | `normal` | **33×39** | 绿色草地 (`moss`) | 13 | 同上 |
| 雪地 (Snow) | `snow` | **33×39** | 白色雪地 (`snow`) | 13 | 同上 |
| 炼狱 (Hell) | `hell` | **33×39** | 熔岩 (`lava`) | 13 | 同上 |

- **每格像素**：32 × 32
- **房间占比**：房间占地图总面积 ≥ 60%
- **泉水**：游戏内**完全移除**泉水系统（猎梦者无泉水回血；出生点为走廊格，绷带自带回血）
- **四边凸出走廊**：每张地图的四条边正中央各有 1 格"凸出走廊格"用于进出地图边界：
  - 顶部 `(18, 0)` / 底部 `(18, 38)` / 左侧 `(0, 19)` / 右侧 `(32, 19)`

> 候选池定义于 `core/config.py: MAP_TYPES = [MAP_TYPE_EASY, MAP_TYPE_NORMAL, MAP_TYPE_SNOW, MAP_TYPE_HELL]`。
> 旧 `MAP_TYPE_HARD` 常量保留为兼容（值 `'hard'`），但已**不再出现在候选池中**。

### 6.2 矩阵值与 Tile 一一对应

4 张地图**全部**采用统一的二维矩阵精确描述，矩阵值直接对应 Tile 类型：

| 矩阵值 | 常量 | 含义 | 渲染 |
|--------|------|------|------|
| `0` | `TILE_EMPTY` | 走廊 | 白色地砖 |
| `1` | `TILE_ROOM` | 房间内部 | 黄色地砖 |
| `2` | `TILE_WALL` | 墙体 | 黑色 |
| `3` | `TILE_DOOR` | 房门（蓝格 / 初始位置） | 蓝色 |
| `4` | `TILE_RED_CHANNEL` | 进门箭头标记（红格）| 红地板 + 绿三角箭头 |
| `5` | `TILE_SPAWN` | 猎梦者出生点 | 绿地板（等同走廊） |

> **内部 Grid 扩展值**：`TILE_BED = 6`（仅游戏运行时使用，不出现在矩阵中）。

### 6.3 矩阵地图规则

1. 地图整体为 **33 列 × 39 行** 的矩形（`grid_width=33, grid_height=39`）
2. **四条边正中间向外凸出 1 格走廊**（猎梦者出生点）：
   - 顶部凸出：`(0, 18)`
   - 底部凸出：`(38, 18)`
   - 左侧凸出：`(19, 0)`
   - 右侧凸出：`(19, 32)`
3. 所有房间必须被**墙体（值 2）**完整包围
4. 每个房间**只能有 1 个房门**（值 3，蓝格）
5. 每个房门的**外侧相邻格子**必须是红色箭头标记（值 4，红格）—— 用于门滑入动画
6. 房间以外、非墙体的区域**全部为走廊**（值 0）
7. 猎梦者出生点（值 5）通常在四边凸出走廊格（由矩阵提供），若矩阵中未标记则兜底使用 4 个分散点
8. 房间内**自动布置床位**（床为 `BLDG_BED`），随机选择空闲地砖放置

矩阵由 [world/config/map_*.json](world/config/) 中的 `matrix` 数组提供，加载时由 [world/map_data.py](world/map_data.py) 的 `_build_from_matrix()` 解析为运行时网格和房间列表。

### 6.4 门滑入动画（**全地图通用**）

> 这是本次重构后**所有 4 张地图都支持**的核心机制。

**机制**：每个房间门（值 3 蓝格）有 1 个相邻红格（值 4）作为"进门箭头"。当人类踏入红格时，**门从蓝格平滑滑入红格**，蓝格变墙，红格变门（变完后变回可通行的 TILE_RED_CHANNEL）。

**实现**：
- 加载阶段（`build_map`）：自动扫描矩阵中相邻的 `3 ↔ 4` 对，生成 `door_pairs` 列表
- 运行阶段（`systems/systems.py:_check_door_transition`）：每帧检测人类踏入红格
- 渲染阶段（`main.py:_close_room_door`）：注册门滑入动画 Tween

**JSON 中不再需要 `door_pairs` / `spawn_points` 顶层字段**——全部由矩阵自动扫描生成。

### 6.5 房间防御机制

猎梦者无法进入房间是因为**墙体阻挡**，需要先破门才能进入追击：

1. 房间外墙必须完整围住（除门位置外不留缺口）
2. 门是唯一入口，必须面向 ≥ 2 格宽的走廊
3. 门 HP 降为 0 后，房间对该猎梦者变为可通行
4. 安全期（门完好时）房间内人类处于保护状态

### 6.6 A* 寻路

- 启发式：曼哈顿距离
- 数据结构：`heapq` 优先队列
- 通行规则：

| 实体 | 走廊 | 房间 | 门（完好） | 门（破） | 红格通道 | 出生点 |
|------|------|------|-----------|---------|----------|--------|
| 猎梦者 | ✓ | ✗ | ✗ | ✓ | ✓ | ✓（等同走廊） |
| 游荡人类 | ✓ | ✓ | ✓ | ✓ | ✓ | ✓ |
| 上床人类 | ✗ | ✗ | ✗ | ✗ | ✗ | ✗ |

> 出生点（`TILE_SPAWN=5`）寻路等同走廊。

## 七、实体系统

### 7.1 Human（人类）

| 属性 | 说明 |
|------|------|
| `gold` | 当前金币（**初始 30**） |
| `state` | `WANDERING(0)` / `BED(1)` / `DEAD(2)` |
| `room_id` | 所属房间 ID |
| `grid_col/row` | 逻辑网格位置 |
| `px/py` | 渲染像素位置（平滑插值） |
| `move_path` | A* 路径 |
| `decision_timer` | AI 决策计时器（仅 AI） |
| `is_under_attack` | 当前帧是否被攻击（头像描边） |

**状态机**：`WANDERING → [到床] → BED → [被秒杀] → DEAD`

### 7.2 DreamHunter（猎梦者，5 级成长）

| 等级 | HP  | ATK | 攻速(次/秒) | 移速(格/秒) |
|------|-----|-----|------------|------------|
| Lv1  | 380  | 8  | 0.6 | 3.5 |
| Lv2  | 600  | 12 | 0.8 | 3.5 |
| Lv3  | 900  | 19 | 1.0 | 3.5 |
| Lv4  | 1300 | 28 | 1.2 | 3.5 |
| Lv5  | 1900 | 40 | 1.5 | 4.0 |

**升级机制**：

- 累计伤害达到阈值 `[0, 350, 1200, 3500, 8000]` 自动升级
- 升级时**回满 HP**、**更新移速**
- 升级到 Lv3 时** BGM 切换为紧张曲目**（`HUNTER_INTENSE_LEVEL = 3`）
- **泉水回血**：已移除（场地无泉水，猎梦者无回血机制；绷带自带回血见 §9）

### 7.3 Building（20 种建筑）

| 分类 | 类型 | 标识 | 功能 | 等级 |
|------|------|------|------|------|
| **基础** | 门 | `BLDG_DOOR (0)` | 阻挡猎梦者 | Lv0-14（**木门 1-5 / 铁门 1-5 / 金门 1-5**，3 材质 × 5 级） |
| | 床 | `BLDG_BED (1)` | 产出金币 | Lv0-5（6 级） |

## 八、战斗与 AI 系统

### 8.1 战斗主流程

`update_combat()` 每帧执行 5 个子任务：

```
1. 猎梦者攻击门（相邻门 HP 扣减 + 累计伤害触发升级）
2. 炮塔自动攻击（射程 + 可达性检测 → 发射子弹）
3. 触碰秒杀（走廊游荡 / 破门房间内）
4. 子弹追踪 + 蛤蟆舌头更新
5. 维修台回血（累积器模式）
```

### 8.2 秒杀规则

| 人类状态 | 位置 | 接触结果 |
|----------|------|----------|
| `WANDERING` | 走廊 | 立即秒杀 |
| `BED` | 门未破的房间 | 不触发（门挡住） |
| `BED` | 门已破的房间 | 立即秒杀 |

### 8.3 AI 人类决策

每 3-5 秒（`AI_DECISION_MIN/MAX`）决策一次，优先级：

```
1. 升级床（最高优先级）
2. 升级门（门等级 < 床等级时）
3. 建造炮塔（房间空位 + 金币 ≥ 55）
4. 建造维修台（有炮塔后 + 金币 ≥ 30）
5. 升级炮塔（炮塔等级 < 床等级时）
```

### 8.4 AI 猎梦者决策

**状态机**：

```
正常攻击 ──[HP ≤ 0]──→ 死亡
```

**目标选择策略**：

1. 优先攻击**有活人入住**且**门未破**的房间
2. 选**门等级最低**的目标（容易破）
3. 门破后进入房间追击**最近可通行格**靠近人类
4. 兜底：所有门完好但都无人时 → 追击走廊游荡者

> 注：旧版的"残血逃跑回泉水"机制已移除。猎梦者血量较低时会持续攻击，依靠**绷带自回血**（HP<30% 自动回血到 50%）维持生存，或被玩家击杀。

### 8.5 门滑入动画流程

```
人类踏入红格 (TILE_RED_CHANNEL)
       ↓
systems/systems.py:_check_door_transition
       ↓
注册门滑入 Tween 动画（SNOW_DOOR_SLIDE_DURATION = 0.35 秒）
       ↓
main.py:_close_room_door
       ↓
门从蓝格 (TILE_DOOR) 平滑滑入红格位置
蓝格变墙 (TILE_WALL)，红格变门 (TILE_DOOR)
       ↓
动画结束后红格恢复为 TILE_RED_CHANNEL（等同走廊）
```

## 九、猎梦者技能

### 9.1 5 种猎梦者总览

| 猎梦者 | HP 倍率 | ATK 倍率 | 独特技能 | 触发 | 效果 |
|--------|---------|----------|----------|------|------|
| **晴天娃娃** | 1.0 | 1.0 | 狂暴 | 攻击 5% 概率 | 1.4× 攻 + 1.3× 速，持续 10s，**冷却 40s** |
| **大头** | 1.0 | 1.5 | 狂暴 | 攻击 8% 概率 | 同上，高攻击 + 狂暴 = 极高爆发 |
| **孙婆婆** | 1.0 | 1.0 | **分身** | 每次升级 | 30% HP / 40% ATK 分身；**15s 后自动合体，主体永久 +5 攻击** |
| **绷带** | 1.5 | 1.0 | **自动回血** | HP < 30% | 30 HP/秒，回血到 50% 停止 |
| **红裙小女孩** | 1.3 | 1.8 | **炮停** | 攻击 6% 概率 | 全图炮塔停火 6s，**冷却 40s** |

### 9.2 通用机制

- **狂暴**（晴天娃娃/大头）：攻击力 ×1.4、攻速 ×1.3，持续 10s，冷却 40s。可被大蒜"恐惧"中断。
- **分身**（孙婆婆）：升级时召唤 `HunterClone`（30% HP / 40% ATK / 80% 攻速），独立寻路攻击门。**15s 后自动寻找主体合体**，合体后**主体永久 +5 攻击力**（每只分身均触发）。分身可被击杀阻止合体加攻击。
- **自动回血**（绷带）：HP < 30% 时持续回血到 50% 停止，是高血量坦克。
- **炮停**（红裙小女孩）：全图炮塔停火 6s，冷却 40s，炮塔密集时威胁极大。
- **残血机制**：泉水系统已移除，猎梦者**不再回泉水**。血量较低时持续战斗，仅绷带可自回血（HP<30% → 50%），其他猎梦者依靠玩家击杀节奏应对。

### 9.3 应对策略

| 猎梦者 | 建议应对 |
|--------|----------|
| 晴天娃娃 / 大头 | 狂暴期需配合大蒜 / 能量罩 / 维修台 |
| 孙婆婆 | 优先击杀分身（屠龙刀 / 蛤蟆 / 冰箱减速），阻止合体加攻击 |
| 绷带 | 持续高 DPS；诱捕网 + 断头台组合 |
| 红裙小女孩 | 大蒜 / 诱捕网干扰，或在其攻击时快速击杀 |

## 十、黑科技建筑

### 10.1 镜子（`BLDG_MIRROR`）

- **常态**：向最近的猎梦者发射**较粗的蓝色光束**（两端连接镜子和猎梦者）
- **金币产出**（按距离）：
  - 猎梦者在 8 格内：**2 金/秒**
  - 猎梦者在 8 格外：**1 金/秒**
- **灼热红光**：5% 概率/秒触发
  - 光束变**红色**，金币产出提升至 **8 金/秒**
  - 对锁定猎梦者每秒造成 5 点伤害
  - 持续 5 秒，冷却 90 秒

### 10.2 诱捕网（`BLDG_TRAP`）

- **触发条件**：猎梦者 **HP < 30%**
- **效果**：定身 2 秒
- **冷却**：10 秒
- 适合对付残血猎梦者（无泉水回血后更易触发）

### 10.3 蛤蟆（`BLDG_FROG`）

- **攻击方式**：**远程舌攻** —— 从蛤蟆位置吐出**粉色舌头**（带亮点），舌头飞行 1.5 倍子弹速度触碰到猎梦者后造成伤害并返回
- **射程**：7 格
- **DPS**：3（每 0.5 秒攻击一次）
- 视觉上可见完整的"吐舌-触敌-收回"动画

### 10.4 屠龙刀（`BLDG_SWORD`）

- **手动激活**：玩家按 `F` 键触发
- **目标选择**：**无距离限制**，优先砍**正在攻击玩家所在房间门**的猎梦者；没有则砍最近的
- **效果**：造成猎梦者 **20% 最大 HP** 伤害
- **冷却**：15 秒
- 能量罩激活时无法对其造成伤害

### 10.5 其他黑科技

| 建筑 | 效果 | 触发 |
|------|------|------|
| **能量罩** | 门无敌 3 秒 | 门血 < 30% |
| **断头台** | 斩猎梦者 10% 最大血 | 猎梦者 HP < 20% |
| **大蒜** | 熏走猎梦者 2 秒 | 门血 < 30% |
| **冰箱** | 猎梦者攻速 -20% | 常驻 |

## 十一、渲染与音频

### 11.1 渲染管线

"**Viewport → Scale**" 流程：

```
世界坐标 (33×39 × 32 px)
  → viewport (固定 416×416 px, 13×13 格)
  → pygame.transform.scale
  → 屏幕游戏区（按窗口尺寸自适应）
```

**分层绘制**：地图 → 建筑 → 实体 → 子弹/舌头 → 屏幕坐标系 HUD/菜单/特效。

### 11.2 程序化纹理

无任何外部图片资源，所有纹理通过 `pygame.draw.*` 实时绘制：

| 元素 | 绘制方式 |
|------|----------|
| 房间地板 | 单格一砖 + 左上高光 + 右下阴影 |
| 走廊地板 | 8 px 横向木板 + 木纹 + 竖向接缝 |
| 门 | 3 形态（**木门 / 铁门 / 金门**），等级越高越精致（铆钉、雕花） |
| 床 | 枕头 + 褶皱被子，尺寸随等级变大 |
| 炮塔 | 底座 + 炮管（Lv2+ 双管，Lv3+ 金色） |
| 蛤蟆 | 主体 + 舌头（粉色线段 + 亮点动画） |
| 镜子 | 镜面 + 蓝色/红色光束（连接到猎梦者） |
| 红格通道 | 红地板 + **绿色三角箭头**（方向指向房间内部，由 4 方向 N/S/E/W 旋转） |

### 11.3 音频系统

[core/audio.py](core/audio.py)：

- **BGM**：优先加载 `assets/audio/bgm/*.ogg`，缺失时回退到程序化生成的环境音
- **SFX**：全部由 `struct` + `math` 程序化生成（零外部音频）
- **音量控制**：HUD 喇叭按钮 / 暂停菜单均可调整 BGM / SFX 独立音量
- **紧张 BGM**：猎梦者升级到 Lv3 后自动切换为 `BGM_BATTLE_INTENSE`
- **关键 SFX**：`door_hit` / `turret_fire` / `bullet_hit` / `human_death` / `hunter_upgrade` / `dragon_sword`

## 十四、关键设计亮点

1. **统一 33×39 矩阵描述**：4 张地图共用同一套矩阵格式 + 同一套 Tile 含义，加载逻辑完全统一
2. **门对 / 出生点自动扫描**：JSON 不再需要 `door_pairs` / `spawn_points` 字段，由 `build_map` 扫描矩阵值 3/4/5 自动生成
3. **门滑入动画通用化**：通过 `TILE_RED_CHANNEL`（值 4）作为触发器，所有 4 张地图自动支持
4. **猎梦者出生点统一**：4 个分散出生点（地图四边正中央的"凸出走廊格"），所有地图使用同一套，杜绝"和人类同位置"bug
5. **房间预订防冲突**：`reserved_room_ids` 集合避免 AI 抢同一张床
6. **双层坐标系**：`grid_col/row` 寻路 + `px/py` 渲染插值
7. **累积器模式**处理 dt：维修台使用 `_repair_accum` 避免整数丢失
8. **AI 与玩家共享规则**：寻路/攻击使用相同的 `find_path` / `_hunter_attack_door`
9. **关卡式 AI 决策冷却**：`random.uniform(3, 5)` 防止同帧决策卡顿
10. **门 HP 动态影响寻路**：`is_walkable_for()` 接收 `door_hp_map`，破门后房间变为可通行
11. **视口渲染 + 屏幕坐标**：先在 416×416 viewport 绘制 → 缩放 → HUD/菜单在屏幕坐标系绘制
12. **数据驱动平衡**：100+ 可调参数集中于 `config.py`，所有升级数据均通过索引访问
